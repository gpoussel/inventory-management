Data Layer

inventory.json
~20 records
SKU, name, warehouse, qty_on_hand, reorder_point, unit_cost, category, location
orders.json
200+ records
order_number, customer, items[], status, dates, total_value, warehouse (Jan–Dec 2025)
demand_forecasts.json
9 records
item_sku, current_demand, forecasted_demand, trend (increasing/stable/decreasing), 30-day period
backlog_items.json
4 records
order_id, item_sku, qty_needed, qty_available, days_delayed, priority (high/med/low)
spending.json
1 summary object
Procurement, operational, labor, overhead totals + 12-month breakdown + 4 category splits + YoY%
transactions.json
50+ records
date, description, category, warehouse, amount, vendor, type (purchase/expense)
purchase_orders.json
0 (runtime only)
Created via POST /api/purchase-orders at runtime, stored in-memory only — resets on restart
Data Notes
Important
All data loaded once at startup into Python lists. No persistence. Filters applied via list comprehension. No DB queries — all in RAM.
